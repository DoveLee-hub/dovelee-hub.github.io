4.1 定义函数

定义方式1

JAVA中：
public 返回值类型 方法名{
  return 返回值：
}

JavaScript中：
function 函数名（变量）{
  函数内容；
}
例如 function abs(x){
  if(x>0){
    return x;
  }
	else{
    return -x;
  }
}

如果没有执行return，函数执行完也会返回结果，结果为undefined

定义方式2

var abs = function{
  if(x>0){
    return x;
  }
	else{
    return -x;
  }
}

两种方法效果一致

function(x){…..}是一个匿名函数，但是可以把结果赋值给abs，通过abs就可以返回调用函数

调用函数

abs(10) //10
abs(-10) //10

4.2 变量的作用域

  • 在JavaScript中，var定义的变量实际是有作用域的 假设在函数体中声明，则在函数体外不可以使用（非要实现的话，后面可以研究一下闭包）

  • 内部函数可以访问外部函数的成员，反之则不行

  • 假设，内部函数变量和外部函数的变量重名，在JavaScript中函数查找变量从自身的函数开始，从内向外查找，假设外部存在同名变量，则内部函数会自动屏蔽外部函数中的变量

  • 提升变量作用域：

    会自动提升y的声明，但不会提升y的赋值

 function a1(){
        var x='x'+y;
        console.log(x);
        var y = 'y';
    }
结果：xundefined

等价于：

 function a2(){
        var y;
        var x='x'+y;
        console.log(x);
        y = 'y';
    }

注：所有的变量声明都放在函数头部，不要乱放，养成好习惯

function a3(){

JavaScript只有一个全局作用域，任何变量（函数也可以视为变量），假设没有在函数作用范围内找到，就会向外查找，如果在全局作用域（window）都没找到，会报错ReferenceError（引用失败）

我们所有的全局变量都会绑定在window上，如果不同的js文件使用了相同的全局变量，就会导致混乱与冲突，如何解决？

</script>

js事件函数中的function(e){}

简单来说就是指向了当前发生的事件（click、mouseover等等），保存了当前事件的信息。如鼠标点击事件，有鼠标的坐标信息。其中，e是标准浏览器传递进去的事件参数，低版本IE不会传递，事件参数放置在window.event对象中，所以兼容低版本的IE要加上这句e=e||window.event;

注：e可加可不加，如果后面需要获取操作的信息 就要加e 如果不需要，只是完成一个事件，就可以不加e
