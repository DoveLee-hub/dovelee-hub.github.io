Java网络编程

1.1 概述

什么是计算机网络

​ 计算机网络是指将地理位置不同的具有独立功能的多台计算机及外部设备，通过通信线路连接起来，在网络操作系统，网络管理软件及网络通信协议的管理和协调下，实现资源共享的信息传递的计算机系统

网络编程的目的

​ 传输交流信息，数据交换，通信

想要达到这效果需要什么

​ 1.如何准确的定位网络上的一台主机： IP地址，端口，定位到这个计算机上的某个资源

​ 2.找到了这个主机，如何传输数据呢？

Javaweb：网页编程 B/S

网络编程：TCP/IP C/S

1.3 IP

IP相关知识

  • 唯一定位一台网络上的计算机

  • 123.0.0.1 本机Localhost

  • IP地址分类

    • IPV4/IPV6
      • IPV4 123.0.0.1 4个字节组成
      • IPV6 2001:0db8:3c4d:0015:0000:0000:1a2f:1a2b 128位 8个无符号整数组成
    • 公网（互联网）-私网（局域网）

  • 域名：记忆IP问题

IP地址：InetAddress类

  • 没有构造方法 只能通过static方法返回InetAddress类的对象

    import java.net.InetAddress;
import java.net.UnknownHostException;
  
public class InetAddress_Test {
    public static void main(String[] args) throws UnknownHostException {
        try {
            InetAddress Address1 = InetAddress.getByName("www.baidu.com");
            System.out.println(Address1);
            System.out.println(Address1.getHostAddress());
            System.out.println(Address1.getHostName());
            System.out.println(Address1.getAddress());
        } catch (UnknownHostException e) {
            e.printStackTrace();
        }
    }
}
#输出
www.baidu.com/220.181.38.149
220.181.38.149
www.baidu.com
[B@1fb3ebeb

1.4 端口

###端口相关知识

端口表示计算机上的一个程序的进程：

  • 不同的进程有不同的端口号 用来区分软件

  • 被规定为0 - 65535

  • 分为TCP端口和UDP端口 范围都为0-65535 单个协议下端口号不可冲突

  • 端口分类

    • 公有端口0～1023

      • HTTP：80
      • HTTPS：43
      • FTP：21
      • Telent:23

    • 程序注册端口：1024～49151 分配用户或程序

      • Tomcat： 8080
      • MySQL： 3306
      • Oracle： 1521

    • 动态、私有 49152-65535

      netstat -ano #查看所有端口
netstat -ano|findstr"端口号"#查看指定端口
tasklist|findstr"端口号" #查看指定端口的进程
#用任务管理器查看端口号

端口：InetSocketAddress类

import java.net.InetSocketAddress;
import java.net.UnknownHostException;

public class InetAddress_Test {
    public static void main(String[] args) throws UnknownHostException {
        InetSocketAddress sockeraddress = new InetSocketAddress("127.0.0.1",8080);
        InetSocketAddress sockeraddress2 = new InetSocketAddress("localhost",8080);
        System.out.println(sockeraddress);
        System.out.println(sockeraddress2);

        System.out.println(sockeraddress.getAddress());
        System.out.println(sockeraddress.getHostName());
        System.out.println(sockeraddress.getPort());
    }
}
输出

##1.5 通信协议

###通信协议相关知识

协议：约定，就好比我们现在说的是普通话

网络通信协议：速率，传输码率，代码结构，传输控制等等

问题：非常复杂

解决办法：大事化小 即分层

TCP/IP协议（实际上是一组协议）

  • TCP：用户传输协议
  • UDP：用户数据报协议
  • IP网络互联协议

TCP/UDP对比

  • TCP：打电话

    • 连接，稳定

    • 三次握手，四次挥手

      #三次握手，最少需要三次，保证稳定连接
A：你瞅啥？
B：瞅你咋的？
A：干一下子！
    
#四次挥手
A：我要走了
B：你真的要走了吗？
B：你真的真的要走了吗？
A：我真的要走了

    • 客户端，服务端

    • 传输完成，释放连接，效率低

  • UDP：发短信

    • 不连接，不稳定
    • 客户端、服务端：没有明确的界限
    • 不管有没有准备好，都可以发给你
    • DDOS：洪水攻击 造成端口堵塞，导致网络崩溃
