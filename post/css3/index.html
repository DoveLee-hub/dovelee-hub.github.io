  • CSS1.0

  • CSS2.0

    • DIV（自定义块）+CSS

  5. 利用SEO，容易被搜索引擎收录

1.5 CSS的三种导入方式

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>

    <!--内部样式-->
    <style>
        h1{
            color: green;
          /*选择器每一行都要有分号结尾*/
        }
    </style>
    <!--外部样式
    通过link进行外部索引
    -->
    <link rel="stylesheet" href="css/style.css">

<!--行内样式：

</html>

注：选择器每一行都要有分号结尾

优先级：后写的会把之前写的覆盖掉，所以行内样式优先级最高，内部样式和外部样式看谁离“被修饰块”近

2.选择器

作用：选择页面上的某一个元素或某一类元素

2.1基本选择器

  1. 标签选择器：选择一类标签

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
   
    <style>
        /*标签选择器，会选择到页面上的所有这个标签的元素*/
    h1{
      color: #00ffff;
    /*在IDE左侧颜色方框处，点击，即可选择更多颜色*/
    }
        p{
            font-size: 80px;
        }
    </style>
</head>
<body>
    <h1>标题1</h1>
    <h1>标题2</h1>
     <p>这是内容</p>
</body>
</html>

  2. 类选择器 class：选择所有class一致的标签，跨标签

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
   
    <style>
        /*类选择器的格式 .class的名称{}
           好处：可以多个标签归类，是同一个class,可以复用
        */
        .DoveLee{
            color: red;
        }
        .MrLee{
            color: green;
        }
    </style>
</head>
<body>
    <!--class名字不能为数字-->
    <h1 class="MrLee">标题1</h1>
    <h1 class="DoveLee">标题2</h1>
    <p class="MrLee">这是内容</p>
</body>
</html>

  3. Id选择器：ID名全局唯一

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
   
    <style>
        /*ID选择器格式    ID必须保证全局唯一且ID名不可为数字
            #id名称{}
        */
        #DoveLee{
            color:red;
        }
    </style>
</head>
<body>
    <h1 ID="DoveLee">标题1</h1>
    <h1 ID="MrLee">标题2</h1>
    <p>这是内容</p>
</body>
</html>

优先级：ID选择器>类选择器>标签选择器

  4. 通用选择器：选择向下所有同辈元素

5.4 父级边框塌陷问题（必考）

子级元素浮动后范围超过了父级元素

比如子级元素是100X100但父级只有50X50

clear: right;右侧不允许有浮动元素
clear: left;左侧不允许有浮动元素
clear：both；两侧不允许有浮动元素
clear：none；

解决方案：

  1. 增加父级元素高度：简单，假设元素有了固定的高度，就会被限制

  2. 增加一个空的div标签，清除浮动：

    简单，但代码中尽量避免空div

<div class="clear"><div>

    clear:both;
    margin:0;
    padding:0;
 }
  1. overflow：简单，但下拉场景避免使用

5.5 对比display和float

  • display

    • 方向不可控

  • float

    • 浮动起来会脱离标准文档流，所以要解决父级边框塌陷问题

6. 定位

6.1 相对定位

相对于原来的位置，进行指定的偏移，相对定位的话，它仍然在标准文档流中，原来的位置会被保留

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
    <style>
        body{
            padding: 20px;
        }
        div{
            margin: 10px;
            padding: 5px;
            font-size: 12px;
            line-height: 25px;
        }
        #father{
            border: 1px solid red;
            padding: 0;
        }
        #first{
            background-color: cyan;
            border: 1px solid red;
            position: relative;
            /*相对定位：上下左右*/
            top: -20px;
            /*距离原来上方-20，所以向上移动*/
            left: 20px;
            /*距离原来左侧20，所以向右移动*/
            /*bottom: 20px;*/
            /*right: 20px;*/
        }
        #second{
            background-color: yellow;
            border: 1px solid red;
        }
        #third{
            background-color: green;
            border: 1px solid red;
        }
    </style>

</head>
<body>
    <div id="father">
        <div id="first">第一个盒子</div>
        <div id="second">第二个盒子</div>
        <div id="third">第三个盒子</div>
    </div>
</body>
</html>

6.2 绝对定位

相对于父级/浏览器的位置，进行指定的偏移，绝对定位的话，它不在标准文档流中，原来的位置不会被保留

  1. 没有父级元素定位的前提下，相对于浏览器定位
  2. 父级元素定位的前提下，相对于父级元素定位
  3. 在父级元素范围内移动
 #second{
            background-color: yellow;
            border: 1px solid red;
            position: absolute;
            /*绝对定位*/
            right: 30px;
        }
